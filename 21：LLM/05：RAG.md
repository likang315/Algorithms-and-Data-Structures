##### 01：RAG（Retrieval Augmented Generation）

- 原理：模型内部学到的“**参数化知识**”（模型权重中固化的的记忆），与来自外部知识库的“**非参数化知识**”（精准、可随时更新的外部数据）相结合。
- 流程（**开卷考试**）：在 LLM 生成前，先通过检索机制从外部知识库中动态获取相关信息，并将这些“参考资料”融入生成过程。

##### 02：技术原理

1. 检索阶段
   - 知识向量化：Embedding 模型，将外部知识 embedding 后，存储向量数据库。
   - 语义召回：查询时，用 Embedding 模型将问题向量化，并通过**相似度搜索（Similarity Search）**，从海量数据中精准锁定与问题最相关的知识。
2. 生成阶段
   - 上下文整合：接收检索阶段送来的相关文档片段以及用户的原始问题，进行有效整合。

###### 评测体系

- 检索相关性
  - 语义准确性
  - 词汇匹配度
- 生成质量

###### 架构演进

- 性能上
  - 索引分层：高频数据启用缓存；
  - 多模态扩展：提升效率和能力边界；
- 架构上
  - 分支模式：并行处理多路检索；
  - 循环模式：进行自我修正；

##### 03：RAG 演进阶段

|          | 初级 RAG（Naive RAG）   | 高级 RAG（Advanced RAG）                             | 模块化 RAG（Modular RAG）                                    |
| :------: | :---------------------- | :--------------------------------------------------- | :----------------------------------------------------------- |
|   流程   | **离线:** `索引`        | **离线:** `索引`                                     | 积木式可编排流程                                             |
|          | **在线:** `检索 → 生成` | **在线:** `...→ 检索前 → ... → 检索后 → ...`         |                                                              |
|   特点   | 基础线性流程            | 增加**检索前后**的优化步骤                           | 模块化、可组合、可动态调整                                   |
| 关键技术 | 基础向量检索            | **查询重写（Query Rewrite）** **结果重排（Rerank）** | **动态路由（Routing）** **查询转换（Query Transformation）** **多路融合（Fusion）** |
|  局限性  | 效果不稳定，难以优化    | 流程相对固定，优化点有限                             | 系统复杂性高                                                 |

##### 04：构键流程

1. **数据准备与清洗**：将Word 等多源异构数据标准化，并采用合理的**分块策略**（如按语义段落切分而非固定字符数）;
2. **索引构建**：将切分好的文本通过**Embedding模型**转化为向量，并存入向量数据库；
3. **检索策略优化**：不要依赖单一的向量搜索。可以采用**混合检索**（向量+关键词）等方式来提升召回率，并引入**重排序**模型对检索结果进行二次精选，确保 LLM 看到的都是精华。
4. **生成与提示工程**：设计一套清晰的 **Prompt 模板**，引导 LLM 基于检索到的上下文回答用户问题。
