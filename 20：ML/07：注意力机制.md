### 注意力机制

------

[TOC]

##### 01：注意力提示

- 非自主性提示：受物体的突出选择的；
- 自主性提示：受认知和意识的控制选择；

###### 注意力机制

- 查询（自主提示）和键（非自主提示）之间的交互形成了**注意力汇聚**； 注意力汇聚有选择地聚合了值（感官输入）以生成最终的输出。

##### 02：注意力汇聚（Attention Pooling）

###### 核心思想

- 在汇聚（整合）一堆信息时，不要一视同仁，而是根据当前任务的需求，**给不同的信息分配不同的“关注度”（权重），最后加权求和**。
- 注意力汇聚的三大关键角色
  - **查询（Query, Q）**：代表**“我当前想找什么”**。比如你在读一句话，Query 就是你当前正在处理的这个词，它带着任务去其他词那里寻找相关信息。
  - **键（Key, K）**：代表**“信息的索引或标签”**。序列中的每个元素都有一个 Key，用来和 Query 进行匹配，看看自己是否重要。
  - **值（Value, V）**：代表**“信息的实际内容”**。如果某个 Key 和 Query 匹配成功（相关性高），那么它对应的 Value 就会被大量提取出来。

###### 工作原理

- **计算评分（打分）**：用当前的 **Query** 去和所有的 **Key** 做匹配（比如计算向量之间的相似度）。匹配度越高，说明这个信息越重要，得分就越高。
- **归一化权重（算概率）**：通过 Softmax 函数，把上一步得到的所有评分转换成总和为 1 的概率分布（即**注意力权重**）。得分高的元素会获得接近 1 的权重，得分低的权重趋近于 0。
- **加权求和（汇聚）**：用算出来的权重，去对所有的 **Value** 进行加权求和。最终加权和，就是注意力汇聚的输出。
  - **一个简单的公式表达就是**：`输出 = Σ (注意力权重 × value)`
  - 权重告诉模型**关注那些，忽略那些**；

###### 平均汇聚和注意力汇聚

- 平均汇聚：求和取平均，认为所有 Key 同等重要；
- 注意力汇聚：模型会根据当前的 Query（比如要预测下一个词，或者做翻译），动态地计算权重求和。

##### 03：注意力评分函数（Attention Scoring Function）

- 计算“查询（Query）”和“键（Key）”之间的**相似度的函数**，计算值也称为**注意力评分（权重）**。

###### 加性注意力（加性评分）

- 当 Quer y和 Key 是不同维度的向量时，**把 Query 和 Key 拼接或相加后，送入一个前馈神经网络**（通常是带 tanh 激活函数的单层 MLP）中进行计算，最终输出一个标量分数。

###### 缩放点积注意力（缩放点积评分）

- 直接**计算 Query 向量和 Key 向量的内积（点积）**。从几何角度看，点积越大，说明两个向量在方向上越一致，相似度就越高。
- 缩放（Scaled）：点积之后，公式中**会除以一个缩放因子（通常是向量维度 d 的平方根）**。这是因为当向量维度 d 很大时，点**积的结果数值会变得极其庞大，经过 Softmax 函数后会导致梯度趋近于 0（梯度消失）**，严重影响模型训练。除以 d 可以把数值拉回到一个稳定的区间，保证训练的平稳。

$$
a(\mathbf{q}, \mathbf{k}) = \frac{\mathbf{q}^\top \mathbf{k}}{\sqrt{d}}
$$

- 它的硬性要求是 **Query 和 Key 的向量维度必须相同**。

##### 04：多头注意力（Multi-Head Attention）

- 目的：让模型**从多个不同的角度（即多个“头”）并行地观察和理解同一段文本**，最后把大家的观察结果汇总起来。
- 不再依赖单一的注意力机制，而是**并行地运行多组注意力机制。每一个“头”都有自己独立的 Query（查询）、Key（键）和 Value（值）参数矩阵**。这意味着每个头都能学习到不同的特征，关注文本中不同类型的依赖关系。

###### 工作原理

1. **线性投影与拆分（分头）**
   - 首先，把输入的 Query、Key、Value 向量分别**通过不同的线性层（全连接层）进行变换**。然后，把这些变换后的向量在维度上拆分成多个“子空间”，也就是分给不同的“头”。
   - 比如**基于512cm 的整根木条的全部信息**，重新加工出 8根全新的、长度为 64cm的精炼木条。(一个 512 维的向量，如果有 8 个头，每个头就负责处理 64 维）
2. **并行计算注意力**
   - 每个头在自己的子空间里，独立地执行我们之前聊过的“缩放点积注意力”计算。此时，不同的头可能会关注到完全不同的信息。
3. **拼接（Concat）**
   - 把所有头计算出来的结果向量，重新拼接（Concat）在一起，恢复成原来的维度。
4. **线性融合（汇总）**
   - 最后，把拼接好的向量再通过一个线性层进行一次整体的融合，得到最终的输出。

##### 05：自注意力（Self-Attention）

- 模型在处理一个词时，**能够同时“回头看”并关注同一句话中所有其他的词**，从而精准地捕捉词与词之间的内在关联和上下文关系。

- 根本区别：它的查询（Query）、键（Key）和值（Value）全部来自于**同一个输入序列**。

  | 维度             | 传统注意力机制 (Cross-Attention)                             | 自注意力机制 (Self-Attention)                                |
  | :--------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
  | 通俗叫法         | “跨序列”注意力                                               | “序列内”注意力                                               |
  | **Q, K, V 来源** | Q 来自**目标序列**（如正在翻译的中文）；K 和 V 来自**源序列**（如原本的英文） | Q、K、V **全部来自同一个序列**（比如都是原本的英文）         |
  | 核心目的         | 解决**两个序列之间的对齐**问题（比如：翻译这个词时，我该看原句的哪里？） | 解决**同一个序列内部的依赖**问题（比如：为了理解这个词，我该参考句子里的谁？） |
  | 对象不同         | “**自己问自己**”（句子内部消化，搞懂上下文）                 | “**拿着目标问源头**”（跨序列交互，搞懂对齐关系）             |

##### 06：位置编码（Positional Encoding）

- 作用：告诉模型每一个词在句子中的顺序。**解决并行处理导致顺序丢失的问题**。
- 原因：传统的 RNN（循环神经网络）是像人一样“从左到右”逐字阅读的，所以它天生就知道谁在前、谁在后。但 Transformer 最大的革命是**并行计算**——它是一瞬间把整句话的所有词同时丢进模型里处理的。
  - **人咬狗 和 狗咬人**，如果没有位置编码，语义截然相反。
- 工作原理：
  1. **生成位置向量**
     - 模型会为句子中的每一个位置（第1个词、第2个词……）生成一个独一无二的数字向量。
  2. **向量相加**
     - 把这个“位置向量”直接加到原本的“词向量”上，**最终输入给模型的 = 词的语义信息 + 词的位置信息**。

###### 编码方式

- 绝对位置编码（正弦/余弦函数法）
  - 用一套固定的数学公式（正弦和余弦波）来生成位置编码。
  - 非常有规律。相邻位置的编码很像，距离远的位置编码差异大，**能处理任意长度的文本**。
- 相对位置编码
  - 灵活，模型能根据任务自己摸索出最好的位置表达。但是较为死板。
